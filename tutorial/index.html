Tutorial

Install

Just include nehan.js, nehan.css in the head.

<link rel="stylesheet" href="/path/to/nehan.css">
<script type="text/javascript" src="/path/to/nehan.js"></script>

Getting started

STEP1 - Create paged-element

First of all, create paged-element that contains all generated pages by Nehan.createPagedElement.

And setting styles of <body> by setStyle that represents the size of each paged-media.

And after setting content text by setContent, paged media is generated asyncronously.

You can hook this generating process by onProgress and onComplete.

var pages = Nehan.createPagedElement().setStyle("body", {
  "flow":"lr-tb", // or "tb-rl"
  "font-size":16,
  // fontSize:16, // camel-case is supported(version >= 5.1.1)
  "width":640, // page width
  "height":480 // page height
}).setContent("<p>hello, nehan.js!!</p>", {
  onProgress: function(tree, ctx){
    // tree.pageNo, tree.percent available
  },
  onComplete: function(time){
    // console.log("finished! %fmsec", time);
  }
});

Camel cased property(like fontSize) is supported since version 5.1.1.

STEP2 - Set paged media to some target.

All paged media are stored in paged-element internally, so if you want to display them, you need to obtain the DOM node that wraps paged media by getElement.

$("#target-element").append(pages.getElement());

STEP3 - Bind events

Paged-element contains all pages and state of current page, so setNextPage and setPrevPage is the method that updates the inner state of current page no of paged-element.

You can also set current page no directly by setPage, and total page count is given by getPageCount.

$(".next").click(function(){ pages.setNextPage() });
$(".prev").click(function(){ pages.setPrevPage() });
$(".first").click(function(){ pages.setPage(0) });
$(".last").click(function(){ pages.setPage(pages.getPageCount() - 1) });

Setting style

Preface

1. logical property

Because nehan.js is logical layout engine, you must use logical property.

For example, generally you might set width of some element like width:300px, but in nehan.js, the meaning of width is different as to flow of document.

If flow of document is lr-tb, width means the size along inline level direction, but if flow is tb-rl, width means the size along block level direction, because in vertical mode, block level flows from left to right(or right to left).

So if you want to set the size of inline direction, you must use the property measure, and use the property extent for the block level direction.

For more details for logical property, see CSS Writing Modes Module Level3 - Abstract box dimemsions..

2. logical direction

For example, lets define some simple style like bellow.

// all class name of nehan is required "nehan-" prefix.
pages.setStyle(".example1", {
  measure:"100px",
  extent:"200px",
  margin:{start:0, end:0, before:0, after:"20px"}
});

You might wonder what is {start:0, end:0, before:0, after:"20px"}?

This is logical direction, start and end are along inline direction, and before and after are along block level direction.

So if flow is lr-tb, margin:{after:"20px"} means margin-bottom:"20px".

But if flow is tb-rl, it means margin-left:20px because in tb-rl, block level flows from right to left.

Example2

Set backgrond-color to blue if nth-child is even, otherwise red.

pages.setStyle(".example2", {
  "background-color":function(ctx){
     if(ctx.getChildIndex() % 2 === 0){
       return "blue";
     } else {
       return "red";
     }
  }
});

Because functional value is supported in nehan.js, you don't have to use css property like nth-child, instead, you can use context object and javascript logic, it's more natual and powerfull isn't it?

Note that type of argument ctx in css-property is SelectorPropContext.

Example3

You can set global style by Nehan.setStyle.

// this is 'local' style for paged-element.
pages.setStyle("p", {
  margin:{after:"1em"}
});

// this is 'global' style.
Nehan.setStyle("h1", {
  "font-size":"3em"
});

// set all width of paged-media to 80% of screen.
Nehan.setStyle("body", {
  "width":function(){
    return Math.floor(screen.width * 80 / 100);
  }
});

Example4

Using onload property, you can change markup dynamically before parsing.

pages.setStyle(".time", {
  onload:function(context){
     var markup = context.getMarkup();
     var now = new Date().toDateString();

     // dynamically rewrite markup content.
     markup.setContent([
       "<p>You have visited this page at <time>" + now + "</time></p>",
       markup.getContent()
     ].join(""));
  }
});

Note that type of argument ctx at onload is NOT SelectorPropContext but SelectorContext.

Unlike SelectorPropContext at css-property, onload is called after all css properties of the target selector are loaded, so compared to SelectorPropContext, SelectorContext has more infomation.

Example5

By using oncreate property, you can hook just after the target selector is converted into DOM.

pages.setStyle("p", {
oncreate:function(context){
    // console.log("abstract logical tree:%o", context.box);
    $(context.dom).click(function(){
       alert("click p!");
    });
  }
});

Note that type of argument ctx at oncreate is DomCreateContext.
